What Is an Object Retexture?

An object retexture replaces the visual sprites inside an existing game object with your own artwork – without touching any of the behaviour scripts (BHAV). You're only changing how it looks, not how it works. This makes it the simplest object modification possible and a good bridge between pure CC creation and full IFF hacking.

✅ What you can change
The game-view sprites (what the object looks like on the lot), the catalogue thumbnail image (what appears in Buy mode), and the speech balloon image (what Sims show when they want to use it).
❌ What you are not changing
Interactions, price, motive effects, skill gains, animations, or any other gameplay behaviour. If you want to change those, see the IFF Hacking page.
🔧 Tools needed
Transmogrifier (to clone the object safely) and IFF Pencil 2 (to open the clone and replace sprites). Both are free from the Sims1Tools GitHub repository and work standalone – no registry fix needed.
🎨 Image editing
GIMP or Photoshop to paint your retexture. Object sprites use Indexed 256-colour BMP, same as body skins. The colour palette in the original sprite file matters – your replacement should match it.

Always clone the original object before editing it. Transmogrifier creates a safe copy of any game object as a new .iff file. Never modify original game files directly – corrupting them can break the game.

The full .iff editor. Opens the cloned object and lets you replace BMP sprites inside it. Handles BMP_, SPR2 and other resource types. Everything you need for a visual-only retexture is in here.

For painting your retexture artwork. You'll export individual sprites from IFF Pencil 2, repaint them, then import them back. Save as Indexed BMP.

Step-by-Step: Retexturing an Object

🚨 Always clone first. Never open and edit an original game .iff file directly. If you corrupt it, you break the object for everyone in your game with no easy way to restore it. Always use Transmogrifier to make a copy first.

Find the object's .iff file

Game objects are in GameData\Objects\. Expansion pack objects are in ExpansionPack1\ through ExpansionPack7\. The filename usually matches the object's name – for example, the standard couch is sofabasic.iff. If you're unsure, use SimShow to browse objects visually.

Clone it with Transmogrifier

Open Transmogrifier, browse to the original .iff file, and click Clone. Give the clone a descriptive name – this becomes the object's filename and internal ID. Save the clone somewhere easy to find, like your Desktop. The clone is completely independent of the original.

Open the clone in IFF Pencil 2

Open IFF Pencil 2 and load your cloned .iff file. In the left panel you'll see all the resources inside the object. Look for BMP_ resources – these are the bitmap sprites. There may also be SPR2 resources for some objects.

Export the sprites you want to replace

Click a BMP_ resource to preview it. Right-click → Export → save as BMP. Do this for each view angle you want to repaint. Objects typically have multiple BMP_ resources – one per zoom level and viewing angle. Common ones are IDs 2000 (catalogue image), 4000 and 6000 (speech bubbles), and higher IDs for the game-view sprites.

Repaint in GIMP

Open the exported BMP in GIMP. The sprite uses Indexed colour – you'll see the colour palette in GIMP's Palettes panel. Paint your retexture using colours from the existing palette for best results. Avoid adding colours not in the palette as this can cause colour-shifting in game. Export as Indexed BMP when done.

Import your new sprites back into IFF Pencil 2

In IFF Pencil 2, right-click the BMP_ resource you exported → Import → select your repainted BMP. Repeat for each sprite you repainted. Save the .iff file.

Place the .iff in Downloads\

Copy your finished .iff into the Downloads\ folder in your Legacy Collection install. Launch the game – the retextured object appears in Buy mode alongside the original.

Test thoroughly

Check the object at all zoom levels (near, medium, far) and from different angles. Object sprites look different depending on how close the camera is – you may need to repaint sprites at different zoom levels if the results look wrong at any distance.

Understanding Object Sprites

What each BMP_ resource does

Objects contain multiple sprite resources. You don't need to replace all of them – just the ones you want to change. Here's what the common ones are:

Resource IDLabel in IFF Pencil 2What it is
BMP_ 2000CatalogThe thumbnail shown in the Buy mode catalogue. Small image, usually 60×60px or similar.
BMP_ 4000Speech (small)The small speech balloon icon shown when a Sim wants to use the object.
BMP_ 6000Speech (large)The large speech balloon icon.
SPR2 100+Sprite groupThe actual game-view sprites at different zoom levels and angles. Multiple resources for near/medium/far views.
ℹ️ Start by replacing just the BMP_ 2000 catalogue image – it's the easiest sprite and a good way to confirm your workflow before tackling the full set of game-view sprites.

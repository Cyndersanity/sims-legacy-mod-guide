Callouts

Four variants. Used for notes, warnings, critical warnings, and tips. Each has a specific emoji that must be used – don't swap them.

Teal – info / note

ℹ️ Note: This is an informational callout. Used for supplementary context, reminders, or things worth knowing that aren't urgent.
<div class="callout teal">ℹ️ <strong>Note:</strong> Your message here.</div>

Gold – warning

⚠️ Warning: This is a warning callout. Used when something could go wrong if the reader isn't careful – wrong folder, common mistake, caveat.
<div class="callout gold">⚠️ <strong>Warning:</strong> Your message here.</div>

Red – critical

🚨 Critical: This is a critical warning. Used sparingly for actions that could break the game, corrupt files, or cause data loss.
<div class="callout red">🚨 <strong>Critical:</strong> Your message here.</div>

Green – tip / confirmation

✅ Tip: This is a green callout. Used for positive confirmations, helpful tips, and "this is working correctly" notes.
<div class="callout green">✅ <strong>Tip:</strong> Your message here.</div>

Expand/collapse toggle (new in v6)

Single button per section. Text toggles between "Expand all" and "Collapse all". Scoped to the closest <section> element so it only affects cards in its own section.

<button class="expand-all" onclick="toggleAll(this)">Expand all</button> // JS (scoped per section): function toggleAll(btn){ var section = btn.closest('section'); var cards = section.querySelectorAll('.ref-card'); var anyOpen = section.querySelector('.ref-card.open'); cards.forEach(function(c){ if(anyOpen) c.classList.remove('open'); else c.classList.add('open'); }); btn.textContent = anyOpen ? 'Expand all' : 'Collapse all'; }

Steps

Auto-numbered using CSS counter. Don't add numbers manually – they increment automatically. Steps use a 1px bottom border between them, removed on the last child.

Third step

Tables

Tables must always be wrapped in .table-wrap. This provides the rounded border, overflow handling, and consistent shadow. Never use a bare <table> without the wrapper.

Standard table

Column AColumn BStatus
Row 1 dataDescription hereWorks
Row 2 dataDescription herePartial
Row 3 dataDescription hereNo
<div class="table-wrap"> <table> <thead> <tr><th>Column A</th><th>Column B</th></tr> </thead> <tbody> <tr><td>Data</td><td>Data</td></tr> </tbody> </table> </div>

Scrollable table (large datasets)

For tables with many rows (e.g. the 52-site active sites list), add a max-height and overflow-y to the wrapper. Use a sticky thead so column headers stay visible while scrolling.

<div class="table-wrap" style="max-height:480px;overflow-y:auto;"> <table> <thead style="position:sticky;top:0;z-index:2;background:var(--cream);"> <tr><th>...</th></tr> </thead> <tbody>...</tbody> </table> </div>

☰ButtonOpens drawer (desktop + mobile)

Desktop hamburger button

A ☰ button sits at the end of the desktop nav bar and opens the same drawer as the mobile "Menu ☰" button. Added because some users find dropdown hover menus awkward. Its CSS class is .nav-ham-desktop. Both the desktop id="navHam" button and the mobile id="navHamMobile" button trigger the same openD() function.

Active states

Add class="nav-link active" to the current page's top-level link. For pages inside a dropdown, add class="nav-dropdown-item active" to the correct item AND class="nav-dropdown-btn active" to the parent button. In the drawer, use nav-drawer-link active or nav-drawer-sublink active.

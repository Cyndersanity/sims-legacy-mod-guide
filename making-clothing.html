Getting the Tools Working

Legacy Collection needs a registry fix before old Maxis tools will run

The Sims Creator, Home Crafter and Art Studio all locate your game via a Windows registry entry that Legacy Collection does not create. You need to add it – either via an installer that does it automatically, or by hand.

⚠️ Home Crafter and Art Studio also need two compatibility settings applied to their .exe files: 16-bit colour mode and Windows XP SP2 compatibility mode. The Sims Creator needs neither. If the installer doesn't apply these automatically, set them manually in each tool's Properties → Compatibility tab.

Route A – Feraligatr Installer Recommended

A community installer that bundles a selection of original Maxis tools (including The Sims Creator, Home Crafter and Art Studio) and handles the registry key and compatibility settings for those tools.

Download the installer

Go to github.com/Secondhand-Feraligatr/Sims-1-Programs-for-Legacy-Collection and download the latest release.

Run and choose Custom Install

Select Custom Install → Select manually to choose which tools to install. They will be placed in a Programs folder inside your Legacy Collection install directory.

If you see a "16-bit colour" error

The installer may not have applied compatibility settings. Right-click the tool's .exe → Properties → Compatibility tab → tick 16-bit colour and set compatibility to Windows XP (Service Pack 2). Applies to Home Crafter and Art Studio – The Sims Creator does not need this.

Route B – Manual Registry Fix

If you prefer to install tools from the Internet Archive and fix the registry yourself, see the CC Troubleshooting – Registry Fix for exact steps. Then apply the compatibility settings above manually.

ℹ️ GIMP and Paint.NET never need a registry fix – they are standalone image editors with no dependency on the game's install location. You can use them to create CC without running any other tool.

How Clothing Works in Sims 1

Very different from later Sims games

⚠️ Sims 1 clothing is nothing like Sims 4. There are no separate tops, bottoms, or hair items – each outfit is a single flat texture painted onto the body mesh. Hair is painted directly onto the head texture. Understanding this first saves a lot of confusion.
👕 One file = the whole outfit
A body skin file contains the Sim's entire appearance – top, bottom, shoes, and hands all in one flat bitmap. To change any part of an outfit you repaint the entire body texture.
🎨 Three skin tones
Sims 1 has light, medium and dark skin tones. If your clothing covers visible skin you need three separate files – one per skin tone – with the matching base skin colour underneath the clothing.
👥 Three adult body types
Skinny (default), fit (athletic), and fat (plus-size). Each comes in male and female – six meshes total for adults. A Sim's body type is permanent – they can only wear clothing that matches their build. A skin painted for a fit female mesh will look distorted on a skinny female body. For widest compatibility, make at least skinny male and skinny female versions.
🧒 Children have their own mesh
Children use a completely separate body mesh. Child clothing files use the prefix Bc rather than Bfa / Bma.
📝

Filename Format

The filename tells the game everything it needs to know

Everyday clothing uses the prefix B (body). The rest of the filename encodes body type, gender, age, skin tone, and a unique outfit name. The format is:

B[body][gender][age] [skin tone]_[outfit name].bmp Example: Bfa fit lgt_RedDress.bmp
SegmentValuesMeaning
First letterBBody – marks this as an everyday clothing skin (goes in the home dresser). See the Terminology page for other prefixes.
Body typefa / ma / caFemale adult / Male adult / Child (age group)
Buildfat / fit / sknFat, fit or skinny body type. Skinny is the default.
Skin tonelgt / med / drkLight, medium or dark base skin. Make one file per tone.
_Outfit nameAny textUnderscore then your outfit name. No spaces – use CamelCase or hyphens.
✅ Minimum viable release: Bfa skn lgt_, Bfa skn med_, Bfa skn drk_ gives a female skinny outfit across all three skin tones. That's 3 files plus their .CMX and .SKN companions (generated by The Sims Creator).

What You Need

The Sims Creator For CMX/SKN files
The official Maxis tool. Provides the 3D body preview, lets you paint on the body template, and exports the .BMP, .CMX and .SKN files the game needs. Install via the Feraligatr installer – see the Getting the Tools Working section above.
GIMP or Photoshop Image editing
For painting your outfit textures. GIMP is free. Body skin BMPs are 256×256 pixels. Critical: you must export as Indexed (256-colour) BMP – the game will show a white blank body otherwise. In GIMP: Image → Mode → Indexed (max 255 colours) → export as .BMP.
↗ gimp.org
Body templates Starting point
Export a body template from The Sims Creator, or extract an existing skin from GameData\Skins\ to paint over. Using an existing skin as a base ensures the UV layout is correct.
Skin Doctor Optional but useful
Validates your skin files before adding to the game. Catches colour depth errors and malformed files. Available from BeyondSims and the Internet Archive.
🚶

Step-by-Step: Making a Clothing Skin

Open The Sims Creator and select a body type

Choose the body type: fat/fit/skinny, female/male, adult/child. Each combination needs its own file. Start with female adult skinny if you're making your first skin.

Export the body template

In The Sims Creator, click the Export button to save the current body texture as a BMP. This gives you the UV layout – the flat image that wraps around the 3D body. Open it in GIMP.

Paint your outfit on top

Work in layers. Keep the base skin on one layer and paint the clothing on top. The front torso sits in the centre of the image; arms are strips to the sides. Pay close attention to seam edges where body parts join – visible seams are one of the most common beginner mistakes.

Create the medium and dark skin tone versions

Swap the base skin layer for the medium skin tone, re-flatten and save. Repeat for dark. All three must have the same outfit painted on top – only the exposed skin area changes colour.

Convert to Indexed 256-colour BMP – this step is critical

GIMP: Image → Mode → Indexed → set Maximum colours to 255 → Convert. Then File → Export As → save as .BMP.
Photoshop: Image → Mode → Indexed Colour → set Colours to 256 → OK. File → Save As → BMP.
Do not save as 24-bit BMP, PNG, JPG or any other format – the game will not read it.

Import back into TSC and export

In The Sims Creator, import your painted BMP. TSC generates the companion .CMX and .SKN files that define the mesh mapping. Export all three files.

Place files in GameData\Skins\

Copy all three file types (.BMP, .CMX, .SKN) directly into GameData\Skins\ in your Legacy Collection install folder. No subfolders – any subfolder and the game won't read them.

Test in game

Launch the game and check the outfit appears in the dresser. If you see a white blank Sim, the BMP wasn't saved as Indexed 256-colour. Go back to step 5.

Custom Meshes – Changing the 3D Shape

Go beyond repainting – create new body shapes, accessories, and outfits

Everything above covers texture-only CC – painting a new look onto an existing body shape. But you can also edit the actual 3D mesh to change how the body, clothing, or accessories look in the game. This is harder and needs a 3D modelling tool, but it lets you make things like flowing dresses, armour, wings, hats, and completely different body shapes.

ℹ️ You don't need to create meshes from scratch. The standard approach is to modify an existing game mesh – stretch it, add geometry, reshape parts. The game's skeleton and animation system stays the same; you're just changing the shape that wraps around it.

How meshes work

Every Sim's appearance involves three files working together, known as the xskin chain:

SKN – the 3D mesh
The 3D shape data – vertices, faces, UV coordinates (how the texture wraps around the shape), and bone assignments (which body part each point follows during animations).
CMX – the config file
A plain text file that tells the game which BMP texture to use on which SKN mesh. You can edit these in Notepad – they're just a few lines long.
BMP – the texture
The flat image (256-colour indexed bitmap) that gets wrapped around the SKN mesh. UV coordinates in the SKN control which part of the image lands where on the 3D shape.
BCF – the skeleton
The bone structure (pelvis, spine, arms, legs, head) that drives animations. You rarely need to touch this – every Sim uses the same skeleton. Your mesh just needs to be weighted to the right bones.

These files live in GameData\Skins\ – some are loose, others are packed inside .FAR archives. Use FARx to extract files from FAR archives before editing.

⚠️ xskin filename rules: The CMX base name segment in an xskin filename must match the CMX filename exactly. Never rename xskin files (filenames starting with xskin-) – the chain will break and the mesh won't load. B-format skin slot numbers are safe to rename.

Modern workflow – Blender + TS1 Blender IO

The easier route if you're starting fresh. Blender is free and TS1 Blender IO by mixiate handles all Sims 1 3D file formats natively.

Install Blender and the TS1 Blender IO addon

Download Blender 4.1+ and the latest TS1 Blender IO release. In Blender, go to Edit → Preferences → Add-ons → Install From Disk, select the downloaded zip. In the addon preferences, set the texture directory to your GameData\Skins\ folder so textures auto-load when importing meshes. See the addon wiki for detailed setup.

Import the skeleton

File → Import → The Sims 1 Skeleton (.bcf). The skeleton files are in GameData\Skins\ (may need to extract from a FAR archive first). This gives you the bone hierarchy that animations use.

Import the mesh you want to modify

Select the skeleton in the viewport, then File → Import → The Sims 1 Mesh (.skn). Choose a base mesh similar to what you want to create – e.g. the default female fit body if you're making a dress. The mesh will load parented to the skeleton with bone weights already assigned and the BMP texture applied.

Edit the mesh

Use Blender's modelling tools – move vertices, extrude faces, sculpt, add geometry. Keep the vertex count reasonable (the game is from 2000) and don't move vertices too far from their original bone assignments or they'll deform oddly during animations.

Edit bone weights if needed

If you've added new geometry (e.g. a skirt extension), assign it to the correct bones using Blender's weight painting tools. Each vertex must be weighted to 1 or 2 bones. The addon wiki has details on the bone naming convention.

UV unwrap and paint the texture

If you changed the mesh shape, you may need to adjust the UV map so your BMP texture wraps correctly. Export the UV layout from Blender and paint the texture in GIMP. Save as 256-colour indexed BMP.

Export the mesh and CMX

File → Export → The Sims 1 Mesh (.skn) and File → Export → The Sims 1 Skin (.cmx). Use the correct xskin naming convention for your filenames.

Place all files in GameData\Skins\ and test

Copy the exported .SKN, .CMX, and your .BMP texture into GameData\Skins\. Launch the game and check the outfit appears correctly in the dresser and animates properly.

Classic workflow – MilkShape 3D

The original method used from 2000–2010. MilkShape 3D ($25 shareware, 30-day trial) has native Sims 1 SKN import/export. Most older tutorials use this tool.

Install MilkShape 3D

Download from milkshape3d.com. It has native SKN import/export – no plugins needed for basic Sims 1 mesh work.

Import the SKN mesh

File → Import → The Sims SKN. Navigate to GameData\Skins\ and choose your base mesh. The skeleton loads automatically – you can toggle it off in the Joints tab (uncheck "Show Skeleton") while editing.

Edit the mesh

Use MilkShape's vertex tools (Select, Move, Rotate, Scale) to reshape the mesh. Work in wireframe mode for precision. Don't add or remove vertices unless you know the bone assignment implications.

Export back to SKN

File → Export → The Sims SKN. Follow the xskin naming convention: xskin-b###fafit_name-PELVIS-BODY.skn where ### is a unique 3-digit number.

Write the CMX file

Create a new text file in Notepad matching your SKN. The CMX format is simple – see the TS1 Blender IO wiki for the exact format, or use SE3D's CMX Builder if you have SimEnhancer 3D.

Paint texture, place files, and test

Same as the Blender workflow – paint your BMP (256-colour indexed), copy SKN + CMX + BMP to GameData\Skins\, test in game.

⚠️ Alternative classic route: If you prefer a 3D editor other than MilkShape (like 3ds Max or an older version of Blender without the TS1 addon), use Skn2Obj to convert SKN → OBJ, edit in your tool, then convert OBJ → SKN. Skn2Obj is available from BeyondSims archives.
